# セットアップ手順書（RYOさん向け）— MEO調査ツール版

このツールを社内で使えるようにする手順です。
プログラミング知識は不要です。所要時間: 約25分。

---

## このツールでできること

- **地域＋業種で検索**: 「五反田 SES」のように入れると、競合がGoogleマップの表示順で一覧表示。各社の星評価・口コミ件数が見える。
- **会社名で検索**: 会社名を入れると、その会社の星評価・口コミ件数・口コミ本文・営業時間が見える。

データはすべてGoogleマップの公式情報（Google Places API経由）です。

---

## 全体の流れ

```
1. Google CloudでPlaces APIを有効化し、APIキーを取得  ← 10分
2. 更新したファイルをGitHubに上書きアップロード        ← 5分
3. VercelにAPIキーを環境変数として登録             ← 3分
4. 再デプロイ                                    ← 2分
5. 動作確認                                      ← 5分
```

**事前に準備するもの**: Googleアカウント、クレジットカード（※下記の注意参照）

---

## ステップ1: Google CloudでPlaces APIキーを取得

### なぜクレジットカードが必要か

Google Maps Platformは、利用にあたってクレジットカード登録が求められます。
ただし**毎月$200分の無料クレジット**があり、このツールの想定利用量なら**実際の請求はほぼ0円**です。
（Places APIのテキスト検索は1000回で約$32。月$200枠 = 月6000回まで実質無料）

「勝手に高額請求される」ことを防ぐため、ステップ1-7で**上限アラート**を設定します。

### 1-1. Google Cloud Consoleにアクセス

ブラウザで https://console.cloud.google.com を開き、Googleアカウントでログイン。

### 1-2. プロジェクトを作成

1. 画面上部の「プロジェクトを選択」をクリック
2. 「新しいプロジェクト」をクリック
3. プロジェクト名: `meo-tool`（任意）
4. 「作成」をクリック
5. 作成したプロジェクトを選択した状態にする

### 1-3. お支払い情報を登録

1. 左上のメニュー（≡）→「お支払い」
2. 「請求先アカウントをリンク」または「請求先アカウントを作成」
3. クレジットカード情報を入力
   - ここで一時的に少額の与信確認がされることがありますが、すぐ戻ります
   - 無料トライアルの案内が出たら有効化してOK

### 1-4. Places APIを有効化

1. 左上メニュー →「APIとサービス」→「ライブラリ」
2. 検索ボックスに `Places API (New)` と入力
3. 検索結果の **「Places API (New)」** をクリック
4. **「有効にする」** ボタンをクリック

> ⚠️ 「Places API」と「Places API (New)」の2つが出ます。**(New)** のほうを有効化してください。このツールは新しいAPIを使います。

### 1-5. APIキーを作成

1. 左メニュー →「APIとサービス」→「認証情報」
2. 上部の「+ 認証情報を作成」→「APIキー」
3. APIキーが生成されます（`AIza...` で始まる文字列）
4. **このキーをコピーしてメモ**しておく（後でVercelに登録します）

### 1-6. APIキーに制限をかける（セキュリティ）

作ったキーの「編集」（鉛筆アイコン）を開いて:

- **APIの制限**: 「キーを制限」を選び、「Places API (New)」だけにチェック
- 「保存」

> これで万一キーが漏れても、Places API以外には使えなくなります。

### 1-7. 予算アラートを設定（請求事故防止）

1. 左メニュー →「お支払い」→「予算とアラート」
2. 「予算を作成」
3. 予算額: `1000円` などの少額に設定
4. アラートのしきい値（50%、90%、100%）で通知が来るようにする

> これで、想定外に課金が増えたらメールで気づけます。

## ステップ2: ファイルをGitHubに上書きアップロード

1. ブラウザで `googlesearch` リポジトリを開く
2. 「Add file」→「Upload files」
3. `C:\自作アプリ\googlesearch` の中から、更新された以下をアップロード:
   - `api`（フォルダごと）
   - `public`（フォルダごと）
   - `README.md`
   - `セットアップ手順書.md`
4. 「You are replacing N existing files」（上書き警告）が出ればOK
5. 「Commit changes」をクリック

→ Vercelが自動で再デプロイを始めます

---

## ステップ3: VercelにAPIキーを環境変数として登録

### 3-1. 古い環境変数を削除（重要）

前のSEOツールで登録した `ANTHROPIC_API_KEY` はもう使いません。

1. Vercel → googlesearch プロジェクト
2. 左サイドバー「Environment Variables」
3. `ANTHROPIC_API_KEY` の行の「⋯」→「Remove」（削除）

### 3-2. 新しい環境変数を追加

1. 「Add Environment Variable」をクリック
2. 入力:
   - **Key**: `GOOGLE_MAPS_API_KEY`
   - **Value**: ステップ1-5で取得したAPIキー（`AIza...`）
   - **Environments**: Production / Preview の両方にチェック
3. 「Save」

---

## ステップ4: 再デプロイ

1. Vercel左サイドバー「Deployments」
2. 一番上のデプロイ行の「⋯」→「Redeploy」
3. 「Use existing Build Cache」のチェックは**外す**
4. 「Redeploy」をクリック
5. 1〜2分待ち、ステータスが「Ready」になれば完了

---

## ステップ5: 動作確認

1. 公開URL `https://googlesearch-beta.vercel.app` を開く
   - 開いたら `Ctrl + Shift + R` でハードリフレッシュ
2. 「地域＋業種で検索」タブで、例えば `五反田 SES` と入力
3. 「競合を調べる」をクリック
4. 数秒で競合一覧が表示される

会社・店舗の星評価と口コミ件数が出ていれば成功です。

## トラブルシューティング

### Q1. 「GOOGLE_MAPS_API_KEYが設定されていません」

ステップ3の環境変数登録、またはステップ4の再デプロイがまだです。

### Q2. 「Places API エラー (403)」が出る

考えられる原因:
- Places API (New) が有効化されていない（ステップ1-4を確認）
- APIキーの制限が厳しすぎる（ステップ1-6でPlaces API New が許可されているか確認）
- お支払い情報が未登録（ステップ1-3を確認）

### Q3. 「Places API エラー (400)」が出る

検索クエリの形式の問題です。「地域名 業種」のように入力してみてください。

### Q4. 検索結果が0件

その地域・業種でGoogleマップに登録された会社がない可能性があります。
業種ワードを変える（「SES」→「システム開発」「IT企業」など）と見つかることがあります。

### Q5. 口コミ本文が出ない

Places APIは1店舗につき最大5件の口コミしか返しません。また、口コミがゼロの会社では何も出ません。
